The web directions conference was amazing this year. Extremely high quality presentations across the board, covering pertinent topics ranging from client side persistence to web security, to "object oriented css". Hats off to John Allsopp, the organizer. But I don't want to write about the conference, I'll leave that to someone else. I wanted to dig into an interesting small phenomenon that happened at the conference: More than half of the talks I attended concluded with a 'Contact Me/More Information' slide that included a reference to project source on GitHub. Not sourceforge.net, and not code.google.com, but GitHub. What the hell is going on?

Maybe it was a fluke. Or maybe not: Ruby On Rails, the high visibility web application development framework offered source on GitHub right when it launched, in April of 2008. YUI announced they would move to GitHub in December of 2008. The Cappuccino "Objective-J" framework is hosted on GitHub. [NOTE: comb github announcements for some of the other popular projects on github]. Why have these projects chosen git? Why have they chosen GitHub?

More news, GitHub won the "Best Bootstrapped Startup of the Year" award on TechCrunch, a community based vote. Let's take it to an emperical popular vote on search.yahoo.com: 585 results for "github rocks", 11 turn up for "github sucks". On google trends, we've got something that looks like exponential growth. Further, it looks like as many people are now interested in git as are interested in subversion in this trend comparison graph. Finally, note the recent increase of interest in git itself seems primarily driven by the interest around GitHub. Git, give the Hub a big kiss. It's safe to call this a frenzy, so again, what the hell is going on? Is all this warranted, or is GitHub a fancy boy band that'll be gone tommorrow? Maybe it's fear, are we pretending to be interested so we don't get called "ugly and stupid" by Linus Torvalds, git's father? (ED: he's done some other interesting stuff, I just can't recall what right now)
